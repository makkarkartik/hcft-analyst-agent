HCFT Agent — Guardrail rings (defense in depth)

Concentric rings, not a single wall: a request is checked on the way in (input ring), around any action it takes (action ring), and on the way out (output ring). Each ring fails independently, so one bypass isn't a full compromise. Agentic ≠ API security — the threats here are prompt injection (incl. via retrieved chunks), the confused deputy, and ungrounded output.

Why rings (defense in depth): fail-closed guards (a guard error blocks) vs fail-open observability (a telemetry error never blocks). Eval/guard duality: every guard has an offline eval twin (e.g. live HHEM groundedness ⇄ offline RAGAS faithfulness).
